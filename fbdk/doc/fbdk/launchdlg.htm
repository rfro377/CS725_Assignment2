Remote Device Launch Dialog
[Return to FBEditor Help]

This dialog box is opened from the Run menu to launch an instance of the RMT_DEV or RMT_FRAME device type in a separate runtime process when a system configuration is being edited. Setting the parameters in this dialog and pressing the OK button is equivalent to executing the following command line in the context of the FBDK main directory:

java -noverify -classpath ./lib;./fbrt.jar; <Type> -n <Name> -s <Socket> -p <Path>

The semantics of parameter ID/value pairs are as shown in the following table. When a parameter is blank in the dialog, the parameter ID/value pair will be missing from the command line and the default value will apply.

Parameter ID Value Default value
-n Sets the device name. (NOTE 1)
-p Adds a list of packages to search for FB classes and data types. (NOTE 2)
-s Sets the management socket to <hostAddress>:<Socket> (NOTES 3,4) 61499

NOTES

  1. The default name is "RDEVICE" for intances of RMT_DEV, and "RMT_FRAME" for instances of RMT_FRAME.
  2. The default packages to be searched for FB classes are fb.rt and its following sub-packages: events; ita; ita.testbed; mach; math; mva; net; plc; plc.bits; plc.comp; plc.math; plc.select; plc.strings; process for RMT_DEV, and additionally hmi for RMT_FRAME. Each package in the comma- or semicolon-separated list of packages will be prepended to this list with the fb.rt. prefix. See the example below.
  3. <hostAddress> is the IP address of the local host, e.g., 169.153.65.1. On the local machine, this may be referred to as localhost or with the default local host IP address 127.0.0.1; when referring to the device from a remote machine, the actual IP address must be used.
  4. When the RMT_DEV or RMT_FRAME instance is successfully launched, an entry for the device appears under the RUN==>Shut Down menu item in the format <deviceName>[<hostAddress>:<Socket>]. This entry can be used to shut down the local device when it is no longer in use, and can also be used to determine the host IP address and socket number for remote access.

Example

For the dialog entries shown above, the equivalent command line would be:

java -noverify -classpath ./lib;./fbrt.jar; RMT_FRAME -n VIEW -s 61500 -p hvac;

  • An instance of the device type fb.rt.hmi.RMT_FRAME will be launched.
  • The device name will be VIEW.
  • FB classes and data types will be found in the fb.rt.hvac package as well as the default packages listed above for the RMT_FRAME type.
  • The device management socket will be at port 61500 of the local host.
  • The entry under the RUN==>Shut Down menu item will look like VIEW[192.168.1.4:61500].
