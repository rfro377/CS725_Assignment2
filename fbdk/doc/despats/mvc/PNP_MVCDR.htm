7. Perform Distribution Design

This system configuration illustrates a distribution design of the pick and place example in which the Model+View functions are located in a remote device named PNP, the Control+HMI functions are in a local device named CONTROL; and the Diagnostic functions are in a local device named DIAGNOSTICS.

Local multicast communications between the PNP device and other devices are replaced with distributed multicast communications using the Tagged Data Design Pattern:

  • A TAGS resource in the PNP device provides mapping between remote and local multicast communication of the sensor and actuator data of the two-dimensional pick-and-place unit as well as the conveyor's PRESENT sensor and the MAGNET actuator state, which are used by both the conveyor and pick-and-place unit models.
  • Similarly, mapping between local and remote multicast of this data is implemented in the TAGS resource of the CONTROL devices, where local multicast of data is shared between the CONTROL and DIAGNOSTICS devices.
