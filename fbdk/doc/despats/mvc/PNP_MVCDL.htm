6. Develop and Test Diagnostics

This system configuration implements the layered Model+View+Control+Diagnostics testing architecture for the pick and place example.

  • The PNP and CONTROL devices encapsulate combined architectural functions in the same manner as in the PNP_MVCL example.
  • The DIAGNOSTICS device implements the Diagnostic functions. These are implemented in the MONITOR resource by four instances of the MECH1_DIAGL type for forward and reverse motion of the x and y axes, respectively. Diagnostic messages are logged in the LOG resource by an instance of the DIAG_LOG type.
  • Inter-layer communication is implemented with the local multicast pattern.
