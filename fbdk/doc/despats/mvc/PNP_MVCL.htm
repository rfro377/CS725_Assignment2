5. Develop and Test Controllers

This system configuration implements the layered Model+View+Control testing architecture for the pick and place example.

  • The Model/View functionality of the PNP devices is the same as in the PNP_MVL configuration.
  • The conveyor control HMI is moved to the CNV_CTL panel in the PNP device, since the control of the conveyor is not the focus of this example.
  • The CONTROL device contains five resources to perform HMI and/or Controller functions:
    1. The MODE_CTL resource utilizes an instance of the MODE_AMS function block type and associated HMI elements to implement the state transitions among AUTO, MANUAL and STOPPED control modes. These modes are broadcast to the other control elements using the local multicast pattern.
      • The STOP button is implemented as an instance of the IMG_BOOL type.
    2. The AUTO_CTL resource utilizes an instance of the PNP_CTL_LD type to implement the control functions required for the automated sequential control of the pick and place system. Sensor values and the AUTO control mode are obtained, and actuator values are broadcast, using the local multicast pattern.
    3. The ACTUATORS resource performs the same HMI functions as in the PNP_MVL configuration, with the following differences:
      • Changes made in the check box inputs are only passed to the actuators when the controller is in the MANUAL mode (except for the MAGNET value, which is passed to its actuator any time it is changed.
      • As a result, the check boxes only indicate the most recent value manually entered and may not show the actual current values of the actuator data.
    4. The ACT_STATE resource shows the current actuator values.
    5. The SENSORS resource performs the same HMI functions as in the PNP_MVL configuration, with the exception of the FAULT indicator.
