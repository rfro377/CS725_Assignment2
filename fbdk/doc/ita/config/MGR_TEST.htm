SYSTEM MGR_TEST

This system configuration provides facilities to test the functionality of the management commands and responses required by the Compliance Profile. The table below shows a typical testing session using this configuration.

Open the DUT (Device Under Test) using the Run option of its pop-up menu.
Launch the test configuration using the Run button on the FBEditor's menu bar (you may need to use desktop controls to make the DUT visible after starting).
Verify that the DUT correctly implements the formula

x2 - y2 = (x + y)*(x - y)

by entering appropriate data into its input field (be sure to follow up each data entry with the Enter key).
Test the QUERY command by entering data as shown and pressing the REQUEST button. The generated command and response are shown in the text area.
Test the Delete Connection command.
Use the QUERY command to verify that the connection is deleted from the data base (to save space, only the textual command and response are shown here). 
<Request ID="3" Action="QUERY" >
  <Connection Source="Y.IND" Destination="*" />
</Request>
<Response ID="3" >
  <EndpointList >
    
  </EndpointList>
</Response>
Verify that the event connection is deleted by entering new data into the DUT (following up each data entry with the Enter key), and observing that the output does not change.
Re-create the connection. 
<Request ID="4" Action="CREATE" >
  <Connection Source="Y.IND" Destination="SUM.REQ" />
</Request>
<Response ID="4" />
Verify that the connection is back in the data base. 
<Request ID="5" Action="QUERY" >
  <Connection Source="Y.IND" Destination="*" />
</Request>
<Response ID="5" >
  <EndpointList >
    SUM.REQ
  </EndpointList>
</Response>
Verify that the connection is re-established in the DUT by re-entering both input data values with the Enter key.
Query the type and state of the SUM FB. 
<Request ID="6" Action="QUERY" >
  <FB Name="SUM" Type="*" />
</Request>
<Response ID="6" >
  <FB Name="SUM" Type="FB_ADD_REAL" />
</Response>
<Request ID="7" Action="QUERY" >
  <FB Name="SUM" Type="FB_ADD_REAL" />
</Request>
<Response ID="7" >
  <FBStatus Status="RUNNING" />
</Response>
Delete the SUM FB and check that it is deleted. 
<Request ID="8" Action="DELETE" >
  <FB Name="SUM" Type="FB_ADD_REAL"/>
</Request>
<Response ID="8" />
<Request ID="9" Action="QUERY" >
  <FB Name="SUM" Type="*" />
</Request>
<Response ID="9" Reason="NO_SUCH_OBJECT"/>
Verify that the FB is deleted by entering new data into the DUT (following up each data entry with the Enter key), and observing that the output does not change.
Re-create the FB and verify that it is present and running. 
<Request ID="10" Action="CREATE" >
  <FB Name="SUM" Type="FB_ADD_REAL" />
</Request>
<Response ID="10" />
<Request ID="11" Action="QUERY" >
  <FB Name="SUM" Type="FB_ADD_REAL" />
</Request>
<Response ID="11" >
  <FBStatus Status="RUNNING" />
</Response>
Re-create the FB's connections and verify some of them. 
<Request ID="12" Action="CREATE" >
  <Connection Source="Y.IND" Destination="SUM.REQ" />
</Request>
<Response ID="12" />
<Request ID="13" Action="CREATE" >
  <Connection Source="SUM.CNF" Destination="DIFF.REQ" />
</Request>
<Response ID="13" />
<Request ID="14" Action="CREATE" >
  <Connection Source="X.OUT" Destination="SUM.IN1" />
</Request>
<Response ID="14" />
<Request ID="15" Action="CREATE" >
  <Connection Source="Y.OUT" Destination="SUM.IN2" />
</Request>
<Response ID="15" />
<Request ID="16" Action="CREATE" >
  <Connection Source="SUM.OUT" Destination="PROD.IN1" />
</Request>
<Response ID="16" />
<Request ID="17" Action="QUERY" >
  <Connection Source="SUM.OUT" Destination="*" />
</Request>
<Response ID="17" >
  <EndpointList >
    PROD.IN1
  </EndpointList>
</Response>
<Request ID="18" Action="QUERY" >
  <Connection Source="X.OUT" Destination="*" />
</Request>
<Response ID="18" >
  <EndpointList >
    DIFF.IN1,SUM.IN1
  </EndpointList>
</Response>
Verify that the FB and its connections are re-established in the DUT by re-entering both input data values with the Enter key.
