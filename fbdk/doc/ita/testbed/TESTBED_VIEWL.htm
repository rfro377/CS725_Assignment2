2. Develop and Test Views

The TESTBED_VIEWL system configuration shown below implements the layered View testing architecture for the Sortation/Inspection testbed. The HMI device encapsulates the functionality of the human interface layer, and the MODEL device encapsulates the functionality of the View layer. The MODEL device contains two instances of the specialized VIEW_PANEL resource type to provide the necessary layout of View elements for the drilling example. Inter-layer communication is implemented with the local multicast pattern, e.g., by using PUBL_3/SUBL_3 for linking the MECH_HMIL/MECH_VIEWL pairs.
