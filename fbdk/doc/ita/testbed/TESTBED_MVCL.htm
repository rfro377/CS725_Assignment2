5. Develop and Test Controllers

The TESTBED_MVCL system configuration implements the layered Model+View+Control testing architecture for the Sortation/Inspection testbed.

  • The MODEL device encapsulates the functionality of both the Model and View layers of the architecture as in the TESTBED_MVL configuration.
  • The CONTROL device contains both the HMI and Controller elements, including the logic for sequencing and interlocking of operations among the modeled physical elements. Control is achieved through instances of the function block types SORT_CTL, FEED_CTL and REJ_CTL. In addition, the CHUTE_ACTUATORS resource contains instances of the CTU_OUT type provide counting and display of successful assemblies accomplished as well as unassembled rings and pegs lost.
  • Inter-layer communication is implemented with the local multicast pattern.
