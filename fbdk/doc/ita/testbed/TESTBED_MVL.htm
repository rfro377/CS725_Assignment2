4. Develop and Test Models

This system configuration implements the layered Model+View testing architecture for the Sortation/Inspection testbed.

The HMI device encapsulates the functionality of the HMI layer of the architecture - specifically the HMI elements needed to read sensor values and write actuator values - while the MODEL device encapsulates both the Model and the View layers.

As in the VIEW device of the TESTBED_VIEWL configuration, the MODEL device contains three instances of the specialized VIEW_PANEL resource type to provide the necessary layout of View elements for the testbed example. In addition, the CHAIN resource contains an FB_SLIDER control to adjust the proportion of pegs to rings injected into the system by the WKPC_EMIT function block instance in the CHAIN_MDL resource.

The MODEL device also contains eight instances of the EMB_RES type, which in turn contain the Model elements for the corresponding Views, as well as logic to simulate the physics of the interactions among Model elements including their sensors and actuators and transfer of workpieces.

  1. The CLK resource contains the simulation clock, whose events are distributed using the local multicast pattern. It also contains additional logic to provide "artificial gravity", i.e., a constant ON signal to the "motors" of the chutes in the PEG_CHUTE and RING_CHUTES resources.
  2. The CHAIN_MDL resource contains an instance of the CNV_MDLL type, corresponding to the chain in the CHAIN view. An instance of the WKPC_EMIT type is used to feed a random selection of pegs and rings to the chain conveyor model.
  3. The SORT_MDL resource contains an instance of the SOL_MDLL type, corresponding to the sorter solenoid in the CHAIN view.
  4. The PEG_CHUTE resource contains an instance of the CNV_MDLL type, corresponding to the CHUTE element of the CHUTES view.
  5. The RING_CHUTES resource contains instances of the CNV_MDLL type, corresponding to the CHUTE element of the CHUTES view.
  6. The BELT_MDL resource contains intances of the CNV_MDLL and CNVG_MDLL types, corresponding to the ASSY_CHUTE and RING_FEED elements of the CHUTES view, respectively.
  7. The ASSY_MDL resource models the peg/ring assembly.
  8. The INSP_MDL resource uses an instance of the WP_SENSE type to model the operation of the inspection station.

Inter- and intra-layer communication is implemented with the local multicast pattern, in particular for the distribution of the simulation clock and the simulation of workpiece transfer with instances of the LOADL and LOADLE types.
